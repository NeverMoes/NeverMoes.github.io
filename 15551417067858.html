内存布局

各个区域可能存在的异常：

是否线程共享：

程序计数器

当前线程执行字节码的行号执行器。

  1. 实现代码的控制流
  2. 当线程切换时，指示运行位置

虚拟机栈

描述 Java 方法执行的内存模型。
方法的调用在内存中抽象为栈帧，每次方法调用都会有栈帧被压入或者弹出。
栈帧结构如下：

  • 局部变量表，存放基本数据结构与对象引用
  • 操作数栈
  • 动态链接
  • 方法出口

可能出现的异常：

  • StackOverFlowError，栈内存大小不允许动态扩容时，栈深度超过允许的最大深度。
  • OutOfMemoryError，栈内存大小允许扩容时，栈申请的内存用尽。

本地方法栈

与虚拟机栈类似，但是是为 Native 方法提供服务的。
同理也会产生 SOF 和 OOM 异常。

堆

主要用于分配对象实例，也是最大的一块内存。因为是 GC 管理的主要区域，也被称为 GC 堆。具体分类在 GC 算法中再详细说明。

方法区

用于存放类，常量，静态变量等不常改变的数据。
虚拟机规范中叫做方法区，而 HotSpot 的具体实现叫做 永久代。
JDK 1.8 之后这个区域被删除了，因为在这之前这个空间是一个固定大小的空间。

元空间

在 Java8 中，永久代已经被移除，元空间的本质和永久代类似，最大的区别在于： 元空间并不在虚拟机中，而是使用本地内存 。 因此， 默认情况下， 元空间的大小仅受本地内存限制。 类的元数据放入 native memory, 字符串池和类的静态变量放入 java 堆中 ， 这样可以加载多少类的元数据就不再由 MaxPermSize 控制, 而由系统的实际可用空间来控制。

常量池

常量池一般是方法区的一部分。

对象创建过程

创建

  1. 类加载检查，遇到 new 命令是先去常量池中找到类的符号引用，如果没有加载过就执行类的加载过程。
  2. 分配内存，即划内存给对象。根据内存是否规整，分配方式有：指针碰撞和空闲列表
    • 指针碰撞，内存规整情况。使用一个分界指针分开用过的内存和没用过的内存。
    • 空闲列表，内存不规则的情况，使用一个列表维护可以使用的内存块。
  3. 初始化零值，将分配到的空间初始化零值。（不是赋初始值）
  4. 设置对象头，对对象进行必要的设置。
  5. init 方法，执行 init 方法初始对象。

对象布局

对象在内存中的布局可以分为三部分：

  • 对象头，对象头一般分为两部分
    • 运行时数据，比如哈希码，GC 分代年龄，锁状态标识。
    • 类型指针，指向类元数据的指针。
  • 实例数据，即类真正存储的有效信息，即实例域。
  • 对齐填充，没什么含义，就是用于内存对齐用的占位数据。

访问定位

创建好对象之后，Java 栈的本地变量表中就存有了 reference 变量。如何使用 referenc 在堆上访问对象呢。一般有两种方式：

  1. 使用句柄，在堆中划出一块空间作为句柄池，其中存有指向对象实例的指针。
  2. 直接指针，reference 直接指向对象。

重点补充

字面量字符串

常量池中最重要的内容是 String 对象。因为 String 分为字面量对象和堆对象。

String str1 = "abcd"; // 字面量对象
String str2 = new String("abcd"); // 堆对象
System.out.println(str1==str2); // false

比如上面这个例子，用双引号表达的字符串是在常量池中创建的，new 的 String 是在堆中的。所以判等结果是 false。
但是 String 对象提供了一个 intern 方法，会在常量池中找到等于该字符串内容引用并返回，如果没有则在常量池上创建。

String s1 = new String("计算机");
String s2 = s1.intern();
String s3 = "计算机";
System.out.println(s2);//计算机
System.out.println(s1 == s2);//false，因为一个是堆内存中的String对象一个是常量池中的String对象，
System.out.println(s3 == s2);//true，因为两个都是常量池中的String对象

还有一个重要的问题就是：String s1 = new String("abc"); 这句话创建了几个对象？
答案是两个。

String s1 = new String("abc");// 堆内存的地址值
String s2 = "abc";
System.out.println(s1 == s2);// 输出false,因为一个是堆内存，一个是常量池的内存，故两者是不同的。
System.out.println(s1.equals(s2));// 输出true
  1. 字符串 "abc" 被放入常量池。
  2. new 了一份字符串 "abc" 对象放入 Java 堆。
  3. 将 s1 指向堆上的 "abc" 对象。

如何排查 OOM 问题

包装类

整数类型的基本类型有几个包装类：Byte,Short,Integer,Long,Character,Boolean。
这五种包装类会在常量池中创建 数值[-128，127] 的缓存。即落在这个范围内的整数指向常量池中的缓存。

注意：浮点类型 Float,Double 没有缓存。

后台运行线程

Hotspot JVM 后台会运行几个系统线程：

  1. 虚拟机线程(VM thread)
    • JVM 达到安全点时出现。执行一些必须在独立线程中执行的操作。
    • 比如：stop-the-world 垃圾回收、线程栈 dump、线程暂停、线程偏向锁（biased locking）解除
  2. 周期性任务线程
    • 负责执行定时器事件（中断）
  3. GC 线程
    • 编译字节码使用
  4. 编译器线程
    • 编译字节码
  5. 信号分发线程
    • 接收发送到 JVM 的信号并调用适当的方法来处理。
