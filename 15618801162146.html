引言

问题：198. House Robber
有一个强盗要入室盗窃，但是他只能偷不相邻的房屋，如何使得偷窃数额最大。
即有一个数组，你不能取下标相邻数，如何取数使得和最大。

找到递归关系

思路：动态规划一般都是可以分解成当前问题和子问题的一种关系。那我们考虑当前值，对于当前值，只有两种状态，要么取要么不取。
用递归关系式表达就是：maxValue(x) = max[maxValue(i-2) + curValue, maxValue(i-1)]
即，如果取当前值，问题转化为当前值 + 对前 i-2 个数求不相邻序列和最大。
如果不取当前值，问题转化问对前 i-1 个数求不相邻序列和最大。
再在这两种可能性中去取结果最大的那个。

还要一个需要明确的是边界。这里比较明显是：maxValue(i)=0，i<0

递归解 (自顶向下)

直接将递归式转化为代码。

public int rob(int[] nums) {
    return rob(nums, nums.length - 1);
}
private int rob(int[] nums, int i) {
    if (i < 0) {
        return 0;
    }
    return Math.max(rob(nums, i - 2) + nums[i], rob(nums, i - 1));
}

递归+缓存解 (自顶向下)

根据之前递归式，可以想象的出来其实是在搜索一棵树。树的大量节点被重复计算了。
所以可以引入一个缓存来避免这种重复计算。

int[] memo;

public int rob(int[] nums) {
    memo = new int[nums.length + 1];
    Arrays.fill(memo, -1);
    return rob(nums, nums.length - 1);
}

private int rob(int[] nums, int i) {
    if (i < 0) {
        return 0;
    }
    if (memo[i] >= 0) {
        return memo[i];
    }
    int result = Math.max(rob(nums, i - 2) + nums[i], rob(nums, i - 1));
    memo[i] = result;
    return result;
}

这里的 memo[i] 就表示到 i 为止可以得到最大值。
这样可以把时间和空间都优化到 O(n)。但是还是会存在过长的递归调用栈。

迭代+缓存 (自底向上)

有了缓存之后其实可以换一个思路，直接使用循环自底向上地算，但是要注意边界，这个从递归优化到迭代过程，是优化动态规划问题的关键。

public int rob(int[] nums) {
    if (nums.length == 0) return 0;
    int[] memo = new int[nums.length + 1];
    memo[0] = 0;
    memo[1] = nums[0];
    for (int i = 1; i < nums.length; i++) {
        int val = nums[i];
        memo[i+1] = Math.max(memo[i], memo[i-1] + val);
    }
    return memo[nums.length];
}

迭代+N个变量 (自底向上)

如果再仔细观察一下，其实我们只用到了memo[i]和memo[i-1]所以其实只需要两个变量就可以了。

public int rob(int[] nums) {
    if (nums.length == 0) return 0;
    int prev1 = 0;
    int prev2 = 0;
    for (int num : nums) {
        int tmp = prev1;
        prev1 = Math.max(prev2 + num, prev1);
        prev2 = tmp;
    }
    return prev1;
}

套路总结

对于一个 DP 问题，
第一步：确定状态转移方程，一般是一个递推关系式。
我们要思考，第 i 个状态和前 i-1 个状态有什么关系。
比如斐波那契是 f(i) = f(i-1) + f(i-2)
以及上面那个 house robber 是 f(i) = max[f(i-2) + A[i], f(i-1)]

第二步：确定边界。
斐波那契中，f(1)=1, f(2)=1
house robber 中 f(i)=0, i<0

第三步：考虑优化。
一般确定递推关系式，就可以直接用递归得出解。但是这种解法一般不可用，因为时间复杂度和空间复杂度都太高了。
一方面可以使用迭代来代替递归来优化空间和时间复杂度。
另一方面，斐波那契和 house robber 的问题中，其实都只用到了 i-1 和 i-2 的状态，空间复杂度都可以优化到 O(1)。

重要：
一般来说我们很难得出一个 f(i) 是可以直接求解，可能要得出一个中间态的状态转移方程，再在其上得出最终解。
所以在很多 DP 的问题中，会引入一个 DP[] 数组来保存求解的一些中间状态，这个 DP 是通过合理拆分子问题得来。如何理解得到这样的 DP[] 数组是求解 DP 问题的关键。
具体可以参考下面的问题。

最长递增非连续序列

问题：300. Longest Increasing Subsequence
注意不需要连续

思路：
先套路一下，思考子问题与当前状态的关系。
用 LIS(i) 表示问题的解，subArr(i) 表示得到 LIS(i) 解的那个最大递增序列。LISEnd(i) 表示 subArr(i) 中最大的那个元素。
那么可以得到 A(i) > LISEnd ? LIS(i-1)+1 : LIS(i-1)
边界为 f(0) = 1, LISEnd(i) = A[0]

如果按照上面那种方式求解，我们会发现无法直接求解 LISEnd。
因为以第k项结尾的 LIS，一定包含着以第 i-1 之前中某项结尾的LIS。
那么引入一个中间状态，DP[i] 表示以 A[i] 结尾的 LIS。
原问题 LIS(i) 转化为求解 max(DP[i]), 0 < i < A.length
中间态的状态转移方程就为：DP[i] = max(DP[j], A[i]>A[j]) + 1, 0<j<i
边界为 DP[0]=1

/*
f(i) = A[i] > LISEnd ? f(i-1)+1 : f(i-1)
f(0) = 1, LISEnd(i) = A[0]
*/

class Solution {
    public int lengthOfLIS(int[] nums) {
        if (nums == null || nums.length == 0) {
            return 0;
        }
        int[] curMaxLen = new int[nums.length];
        curMaxLen[0] = 1;
        int maxLen = 1;
        for (int i=1; i<nums.length; i++) {
            // 这个值即是 f(i-1)
            int maxEndLen = 0;
            // 搜索 LISEnd 的过程
            for (int j=0; j<i; j++) {
                if (nums[i] > nums[j]) {
                    maxEndLen = Math.max(maxEndLen, curMaxLen[j]);
                }
            }
            curMaxLen[i] = maxEndLen + 1;
            maxLen = Math.max(maxLen, curMaxLen[i]);
        }
        return maxLen;
    }
}

时间复杂度：O(n2)
空间复杂度：O(n)

优化：这里存在一个前向搜索 LISEnd 的过程，这个搜索的过程可以使用二分法进行搜索，可以把时间复杂度优化到 O(nlogn)。

最大连续子序列和

问题：53. Maximum Subarray

思路：
先来拆分子问题，用 DP[i] 表示以第 i 个元素结尾的可能最大子序列和。
那么 maxSubArray(i) = max(DP[0], ..., DP[i])
递推式： DP[i] = A[i] + DP[i-1] > 0 ? DP[i-1] : 0
边界 DP[0] = A[0]

class Solution {
    public int maxSubArray(int[] nums) {
        int[] dp = new int[nums.length];
        dp[0] = nums[0];
        int max = dp[0];
        for(int i = 1; i < nums.length; i++){
            dp[i] = nums[i] + (dp[i-1] > 0 ? dp[i-1] : 0);
            max = Math.max(dp[i], max);
        }
        return max;
    }
}

时间复杂度：O(n)
空间复杂度：O(n)

最大连续子序列之积

问题：152. Maximum Product Subarray

思路：拆分子问题，找到可以与 A[i] 进行递推的关系式。如果序列都是正的，那么解就很简单了，DP[i] 表示以 i 结尾连续子序列的最大积，DP[i]=A[i]*DP[i-1]。最终解就是 max(DP[0],...,DP[A.length-1])
但是要考虑到可能存在 [2,3,-1,-2] 这样的序列，所以还需要保留到 i 为止的最小积。
所以定义 maxEnd 为 i 结尾连续子序列的最大积，minEnd 为 i 结尾连续子序列的最小积。
maxEnd(i) = max(A[i]\*maxEnd[i-1], A[i]\*minEnd[i-1])
minEnd(i) = min(A[i]\*maxEnd[i-1], A[i]\*minEnd[i-1])
再考虑一下乘以零的情况：
maxEnd(i) = max(A[i], max(A[i]\*maxEnd[i-1], A[i]\*minEnd[i-1]))
minEnd 同理。

class Solution {
    public int maxProduct(int nums[]) {
        if (nums == null || nums.length == 0) {
            return 0;
        }
        
        int[] maxEnd = new int[nums.length];
        int[] minEnd = new int[nums.length];
        maxEnd[0] = nums[0];
        minEnd[0] = nums[0];
        int max = nums[0];
        
        for (int i=1; i<nums.length; i++) {
            maxEnd[i] = Math.max(nums[i], Math.max(nums[i]*maxEnd[i-1], nums[i]*minEnd[i-1]));
            minEnd[i] = Math.min(nums[i], Math.min(nums[i]*maxEnd[i-1], nums[i]*minEnd[i-1]));
            max = Math.max(maxEnd[i], max);
        }
        
        return max;
    }
}

时间复杂度：O(n)
空间复杂度：O(n)，这个可以优化到 O(1)

优化：很明显可以看出来其实最大最小值都只用了之前一个节点的状态，所以不需要一个数组来保存，可以简单讲空间复杂度优化到 O(1)。
