Java 基础

基本类型

浮点默认为 Double 类型。
float a = 1.0; 是非法的。

==, equal, hashcode

  • ==，比较值
    • 值类型，直接比较值是否相同
    • 引用类型，比较引用地址是否相同
  • equals，比较内容
    • 默认实现
      • 先比较地址
      • 在比较地址的内容
    • 重写实现，必须要遵守：
      1. 自反性：x.equals(x)必须返回true。
      2. 对称性：x.equals(y)与y.equals(x)的返回值必须相等
      3. 传递性：x.equals(y)为true，y.equals(z)也为true，那么x.equals(z)必须为true。
      4. 一致性：如果对象x和y在equals()中使用的信息都没有改变，那么x.equals(y)值始终不变。
      5. 非空性：x不是null，y为null，则x.equals(y)必须为false。
      6. 覆盖equals时一定要覆盖hashCode

  • hashCode，返回对象 hash 值。

    • 默认实现，默认实现为 JVM 特定的
      • 对象在内存中的地址作为哈希码
    • 重写实现
      1. 把某个非零的常数值，比如17，保存在一个int型的result中。
      2. 对于每个关键域f（equals方法中设计到的每个域），作以下操作。
        1. 为该域计算int类型的散列码
        2. i.如果该域是boolean类型，则计算(f?1:0), ii.如果该域是byte,char,short或者int类型,计算(int)f,
          iii.如果是long类型，计算(int)(f^(f>>>32)).
          iv.如果是float类型，计算Float.floatToIntBits(f).
          v.如果是double类型，计算Double.doubleToLongBits(f),然后再计算long型的hash值
          vi.如果是对象引用，则递归的调用域的hashCode，如果是更复杂的比较，则需要为这个域计算一个范式，然后针对范式调用hashCode，如果为null，返回0
          vii. 如果是一个数组，则把每一个元素当成一个单独的域来处理。i.如果该域是boolean类型，则计算(f?1:0),
          ii.如果该域是byte,char,short或者int类型,计算(int)f,
          iii.如果是long类型，计算(int)(f^(f>>>32)).
          iv.如果是float类型，计算Float.floatToIntBits(f).
          v.如果是double类型，计算Double.doubleToLongBits(f),然后再计算long型的hash值
          vi.如果是对象引用，则递归的调用域的hashCode，如果是更复杂的比较，则需要为这个域计算一个范式，然后针对范式调用hashCode，如果为null，返回0
          vii. 如果是一个数组，则把每一个元素当成一个单独的域来处理。
      3. 返回 result = 31 * result + c。
      4. 编写测试，使得所有相等的实例都有相等的散列码。
  1. hashCode()在哈希表中起作用，如java.util.HashMap。
  2. 如果对象在equals()中使用的信息都没有改变，那么hashCode()值始终不变。
  3. 如果两个对象使用equals()方法判断为相等，则hashCode()方法也应该相等。
  4. 如果两个对象使用equals()方法判断为不相等，则不要求hashCode()也必须不相等；但是不相等的对象产生不相同的 hashCode 可以提高哈希表的性能。

不需要覆写 equals 的情况：

  1. 类的每个实例本质上是唯一的：强调活动实体的而不关心值得，比如Thread，我们在乎的是哪一个线程，这时候用equals就可以比较了。
  2. 不关心类是否提供了逻辑相等的测试功能：有的类的使用者不会用到它的比较值得功能，比如Random类，基本没人会去比较两个随机值吧
  3. 超类已经覆盖了equals，子类也只需要用到超类的行为：比如AbstractMap里已经覆写了equals，那么继承的子类行为上也就需要这个功能，那也不需要再实现了。
  4. 类是私有的或者包级私有的，那也用不到equals方法：这时候需要覆写equals方法来禁用它：@Override public boolean equals(Object obj) { throw new AssertionError();}

参考：面试官爱问的equals与hashCode

构造函数

构造函数不能被继承。只有一个特例：在调用子类的无参构造函数之前，会先调用父类的无参构造函数。
子类可以通过 super 调用父类构造函数，但是这个关键字必须是第一个语句。

继承

子类会继承父类的所有成员，包括私有成员，只是不能调用。

接口与抽象类

  • 方法
    • 接口方法默认 public,abstract
    • 抽象类可以有抽象和非抽象方法，修饰符都可以用
  • 变量
    • 接口中的变量默认为 final
    • 抽象类都可以

&,&&,|,||

  • && 逻辑运算时表示短路与
  • | 逻辑运算时表示非短路或，还表示位运算或
  • || 逻辑运算时表示短路或
