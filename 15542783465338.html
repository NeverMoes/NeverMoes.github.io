全局

数据库

  • SELECT 1 选择数据库
  • FLUSHALL 清空数据库
  • DBSIZE 返回数据库键的个数
  • AUTH password 验证密码是否正确
  • PING 查看服务是否运行

String 字符串

很多操作都是对称或者类似的，只对关键性的部分做说明。

  • 前缀或后缀

    • NX 后缀，表示键不存在时才会生效。
    • B 前缀，表示阻塞操作

  • 键值设置和获取

    • SET/SETNX/GET key value， 设置/获得键值
    • GETSET key value ，设置新值返回旧值
    • SETRANGE/GETRANGE key offset value ， 区间
    • SETBIT/GETBIT key offset value
    • MSET/MSETNX/MGET key value [key value...] ，批量

  • 设置时间

    • SETEX/PSETEX key seconds/milliseconds value ，设置到期时间，秒/毫秒单位/
    • EXPIRE/PEXPIRE key seconds/milliseconds ， 指定键多久后过期
    • EXPIREAT/PEXPIREAT key timestamp 设置键的过期时间
    • TTL/PTTL key ， 获取剩余时间的到期键，毫秒单位
    • PERSIST key ， 移除过期的键

  • 键的操作

    • EXISTS key ， 是否存在
    • DEL key ， 如果存在删除键
    • STRLEN key，返回键的长度
    • KEYS pattern ，正则匹配需要的键
    • RANDOMKEY ， 返回随机键
    • RENAME/RENAMENX key newkey ， 重命名键
    • TYPE key 返回存储在键的数据类型的值
    • MOVE key db ， 移动键到另一个数据库

  • 值的操作

    • DUMP key ， 返回键的值的序列化
    • APPEND key value ，追加值
    • INCR/INCRBY key ， 自增或指定值
    • DECR/DECRBY key

List 列表

  • 增
    • LPUSH/RPUSH/LPUSHX/RPUSHX key value1 [value2]，在列头尾压入
    • LINSERT key BEFORE|AFTER pivot value ，在列表某个元素前或后插入一个元素
  • 删
    • LREM key count value，按索引中删除
    • LPOP, RPOP key ，返回第一个元素
    • LTRIM key start stop ，删除到指定大小
    • BLPOP, BRPOP key1 [key2 ] timeout ，如果为空，以 timeout 阻塞直到可用
    • RPOPLPUSH/BRPOPLPUSH ource destination，从一个列表中弹出一个元素并推到另一个列表
  • 改
    • LSET key index value 按照索引设置值
  • 查
    • LINDEX key index ， 索引返回元素
    • LLEN key ， 返回长度

Hash 字典

  • HSET/HMSET/HSETNX key field value [field value ...] 设置对象指定域的值
  • HGET/HMGET/HGETALL key field [field ...] 获取对象中该/多个/所有域的值
  • HDEL key field[field...] 删除域，不存在的则忽略
  • HEXISTS key field 是否存在域
  • HINCRBY/HINCRBYFLOAT key field value 域的值增加value，原子自增操作，限定integer/float
  • HKEYS/HVALS key 键的所有域/值
  • HLEN key 域的数量
  • HSTRLEN key field 某个域的字符长度
  • HSCAN key cursor [MATCH pattern] [COUNT count] 类似SCAN命令

Set 集合

  • SADD key member [member ...] 添加元素
  • SREM key member [member ...] 删除元素，不存在则忽略
  • SPOP key [count] 获取并删除元素
  • SCARD key 元素数量
  • SDIFF/SDIFFSTORE [destination] key [key ...] [存储]差集
  • SINTER/SINTERSTORE [destination] key [key ...] 交集
  • SUNION/SUNIONSTORE key [key ...] 并集
  • SISMEMBER key member 判断元素存在
  • SMEMBERS key 获取所有元素
  • SMOVE source destination member 移动元素
  • SRANDMEMBER key [count] 随机元素
  • SSCAN key cursor [MATCH pattern] [COUNT count] 迭代元素

Sorted Set 有序集合

有序集合的每个成员会映射一个分数。

  • ZADD key score1 member1 [score2 member2]
  • ZREM key member [member ...]
  • ZSCORE key member 返回成员分数
  • ZCARD key 得到的有序集合成员的数量
  • ZUNIONSTORE destination numkeys key [key ...] 交集
  • ZINTERSTORE destination numkeys key [key ...] 并集
  • ZINCRBY key increment member 成员按值递增
  • ZCOUNT/ZLEXCOUNT key min max 指定分数/字典区间的成员数
  • ZRANK/ZREVRANK key member 返回指定成员的索引，升序/降序
  • ZRANGE/ZRANGEBYLEX/ZRANGEBYSCORE key start stop [WITHSCORES] 索引/字典/分数区间返回，分数返回会升序排序
  • ZREMRANGEBYRANK/ZREMRANGEBYSCORE/ZREMRANGEBYLEX key min max

发布订阅

  • SUBSCRIBE/UNSUBSCRIBE channel [channel ...] 订阅/退订频道
  • PSUBSCRIBE/ PUNSUBSCRIBE pattern [pattern ...] 订阅/退订指定匹配模式的频道
  • PUBSUB subcommand [argument [argument ...]] 查看发布订阅状态

事务

  • MULTI 开始事务
  • EXEC 触发事务
  • DISCARD 取消事务
  • WATCH key [key ...] 如果事务执行前 key 被改变，则打断事务。
  • UNWATCH

备份

  • save, bgsave 快照备份，后者会在后台执行。
    • 会忽略过期的键。

脚本

redis 使用 lua 语言作为脚本语言

EVAL script numkeys key [key ...] arg [arg ...] 执行脚本。
EVALSHA sha1 numkeys key [key ...] arg [arg ...] 执行脚本。
SCRIPT EXISTS script [script ...] 查看指定的脚本是否已经被保存在缓存当中。
SCRIPT FLUSH 从脚本缓存中移除所有脚本。
SCRIPT KILL 杀死当前正在运行的 Lua 脚本。
SCRIPT LOAD script 将脚本 script 添加到脚本缓存中，但并不立即执行这个脚本。

比如

redis 127.0.0.1:6379> EVAL "return {KEYS[1],KEYS[2],ARGV[1],ARGV[2]}" 2 key1 key2 first second

1) "key1"
2) "key2"
3) "first"
4) "second"

配置

# redis进程是否以守护进程的方式运行，yes为是，no为否(不以守护进程的方式运行会占用一个终端)。
daemonize no
# 指定redis进程的PID文件存放位置
pidfile /var/run/redis.pid
# redis进程的端口号
port 6379
# 绑定的主机地址
bind 127.0.0.1
# 客户端闲置多长时间后关闭连接，默认此参数为0即关闭此功能
timeout 300
# redis日志级别，可用的级别有debug.verbose.notice.warning
loglevel verbose
# log文件输出位置，如果进程以守护进程的方式运行，此处又将输出文件设置为stdout的话，就会将日志信息输出到/dev/null里面去了
logfile stdout
# 设置数据库的数量，默认为0可以使用select <dbid>命令在连接上指定数据库id
databases 16
# 指定在多少时间内刷新次数达到多少的时候会将数据同步到数据文件
save <seconds> <changes>
# 指定存储至本地数据库时是否压缩文件，默认为yes即启用存储
rdbcompression yes
# 指定本地数据库文件名
dbfilename dump.db
# 指定本地数据问就按存放位置
dir ./
# 指定当本机为slave服务时，设置master服务的IP地址及端口，在redis启动的时候他会自动跟master进行数据同步
slaveof <masterip> <masterport>
# 当master设置了密码保护时，slave服务连接master的密码
masterauth <master-password>
# 设置redis连接密码，如果配置了连接密码，客户端在连接redis是需要通过AUTH<password>命令提供密码，默认关闭
requirepass footbared
# 设置同一时间最大客户连接数，默认无限制。redis可以同时连接的客户端数为redis程序可以打开的最大文件描述符，如果设置 maxclients 0，表示不作限制。当客户端连接数到达限制时，Redis会关闭新的连接并向客户端返回 max number of clients reached 错误信息
maxclients 128
# 指定Redis最大内存限制，Redis在启动时会把数据加载到内存中，达到最大内存后，Redis会先尝试清除已到期或即将到期的Key。当此方法处理后，仍然到达最大内存设置，将无法再进行写入操作，但仍然可以进行读取操作。Redis新的vm机制，会把Key存放内存，Value会存放在swap区
maxmemory<bytes>
# 指定是否在每次更新操作后进行日志记录，Redis在默认情况下是异步的把数据写入磁盘，如果不开启，可能会在断电时导致一段时间内的数据丢失。因为redis本身同步数据文件是按上面save条件来同步的，所以有的数据会在一段时间内只存在于内存中。默认为no。
appendonly no
# 指定跟新日志文件名默认为appendonly.aof
appendfilename appendonly.aof
# 指定更新日志的条件，有三个可选参数 - no：表示等操作系统进行数据缓存同步到磁盘(快)，always：表示每次更新操作后手动调用fsync()将数据写到磁盘(慢，安全)， everysec：表示每秒同步一次(折衷，默认值)；
appendfsync everysec

其他

1 BGREWRITEAOF
异步执行一个 AOF（AppendOnly File） 文件重写操作
2 BGSAVE
在后台异步保存当前数据库的数据到磁盘
3 CLIENT KILL [ip:port] [ID client-id]
关闭客户端连接
4 CLIENT LIST
获取连接到服务器的客户端连接列表
5 CLIENT GETNAME
获取连接的名称
6 CLIENT PAUSE timeout
在指定时间内终止运行来自客户端的命令
7 CLIENT SETNAME connection-name
设置当前连接的名称
8 CLUSTER SLOTS
获取集群节点的映射数组
9 COMMAND
获取 Redis 命令详情数组
10 COMMAND COUNT
获取 Redis 命令总数
11 COMMAND GETKEYS
获取给定命令的所有键
12 TIME
返回当前服务器时间
13 COMMAND INFO command-name [command-name ...]
获取指定 Redis 命令描述的数组
14 CONFIG GET parameter
获取指定配置参数的值
15 CONFIG REWRITE
对启动 Redis 服务器时所指定的 redis.conf 配置文件进行改写
16 CONFIG SET parameter value
修改 redis 配置参数，无需重启
17 CONFIG RESETSTAT
重置 INFO 命令中的某些统计数据
18 DBSIZE
返回当前数据库的 key 的数量
19 DEBUG OBJECT key
获取 key 的调试信息
20 DEBUG SEGFAULT
让 Redis 服务崩溃
21 FLUSHALL
删除所有数据库的所有key
22 FLUSHDB
删除当前数据库的所有key
23 INFO [section]
获取 Redis 服务器的各种信息和统计数值
24 LASTSAVE
返回最近一次 Redis 成功将数据保存到磁盘上的时间，以 UNIX 时间戳格式表示
25 MONITOR
实时打印出 Redis 服务器接收到的命令，调试用
26 ROLE
返回主从实例所属的角色
27 SAVE
同步保存数据到硬盘
28 SHUTDOWN [NOSAVE] [SAVE]
异步保存数据到硬盘，并关闭服务器
29 SLAVEOF host port
将当前服务器转变为指定服务器的从属服务器(slave server)
30 SLOWLOG subcommand [argument]
管理 redis 的慢日志
31 SYNC
用于复制功能(replication)的内部命令
